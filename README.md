## 我为什么要做这个

市面上有大量的 Amazon 选品工具（Helium 10、Jungle Scout、SellerSprite 等），它们能告诉你**数据是什么**，但很少回答**数据意味着什么**。

比如：工具能告诉你某个竞品有多少条评论，但不会告诉你：

- 评论数和 BSR 排名的统计关系是什么？
- 多少评论才算"够了"？
- 有没有品类是评论数根本不重要的？

这些问题需要统计分析才能回答。而我恰好会。

## 我的分析方法

每篇分析都走同一个流程：

```
选题研究 → 方法论设计 → 数据采集 → 统计分析 → 质量审查 → 成文发布
```

| 阶段 | 做什么 |
|------|--------|
| **选题** | 从 Reddit / LinkedIn 的卖家社区里找大家真正关心的问题 |
| **方法论** | 设计统计方案：用什么检验？检验什么假设？需要什么数据？ |
| **数据采集** | 写爬虫，从 Amazon 页面直接抓数据 |
| **分析** | 探索性分析 → 统计检验 → 可视化 → 找"反直觉"的发现 |
| **审查** | 逐条验证数据声明，审计统计方法，评估表达质量 |
| **成文** | Notebook + 图表 + 说人话的结论，兼顾技术严谨和可读性 |
